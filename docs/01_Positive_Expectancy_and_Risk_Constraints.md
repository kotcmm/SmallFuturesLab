# 建立正期望与风险约束

## 1. 文档目的

本文档解决两个问题：

1. 一套交易规则在数学上怎样才有长期正期望；
2. 小资金账户应该如何从正期望模型反推风险约束。

---

## 2. 核心思想

SmallFuturesLab 的第一步是：

> 先定义正期望条件，再定义风险边界。

需要先确定：

1. 胜率、盈亏比和成本之间的关系；
2. 单笔亏损、每日亏损和连续亏损的上限；
3. 账户在亏损序列中的生存能力。

---

## 3. 可控制与不可控制

交易结果由可控制部分和不可控制部分共同决定。

不可控制部分不能靠意愿解决，只能通过规则承受。

可控制部分必须写成硬约束，并严格执行。

### 3.1 不可控制部分

| 项目 | 说明 |
|---|---|
| 单笔交易结果 | 下一笔交易赚钱还是亏钱不可提前确定。 |
| 短期胜率 | 连续几天、几十笔交易的胜率可能大幅波动。 |
| 连续亏损 | 正期望系统也可能出现连续亏损。 |
| 极端波动 | 突发波动可能导致滑点扩大。 |
| 成交价格 | 实际成交价可能偏离计划价。 |
| 市场状态 | 市场可能长时间不提供适合系统的机会。 |

这些变量只能估计，不能控制。

### 3.2 可控制部分

| 项目 | 约束方向 |
|---|---|
| 是否交易 | 不满足条件就不交易。 |
| 单笔风险 | 每笔亏损限制在预设 R 内。 |
| 仓位手数 | 根据账户权益和止损距离计算。 |
| 成本上限 | 成本超过阈值则放弃。 |
| 每日亏损 | 达到上限后停止。 |
| 连续亏损 | 达到暂停条件后停止。 |
| 最大回撤 | 达到警戒线后降风险，达到停止线后停用。 |
| 执行记录 | 每笔交易必须记录结果。 |

系统建设原则：

> 不试图控制市场，只控制自己暴露给市场的风险。

---

## 4. R 的定义

系统统一使用 `R` 表示风险单位。

这里需要区分两个概念：

| 名称 | 含义 | 来源 |
|---|---|---|
| `AccountR` | 账户允许的单笔风险上限 | 账户权益和单笔风险比例计算出来 |
| `TradeR` | 某一笔交易的计划风险 | 入场价、止损价、合约乘数、手数和成本计算出来 |

账户层风险上限：

```text
AccountR = 账户权益 × 单笔风险比例
```

单笔交易计划风险：

```text
TradeR = |入场价 - 止损价| × 合约乘数 × 手数 + 预估交易成本
```

硬约束：

```text
TradeR <= AccountR
```

期望公式里的 `1R` 指的是 `TradeR`，也就是该笔交易自己的计划风险。

平均亏损倍数 `a` 的定义见：[实际平均亏损](01_Positive_Expectancy_and_Risk_Constraints/Actual_Average_Loss.md)。

```text
a = 实际平均亏损 / TradeR
```

如果止损执行稳定，长期平均亏损应该接近：

```text
a = 1
```

如果 `a` 长期大于 `1`，说明止损执行或成本控制有问题。

---

## 5. 正期望模型

基础公式：

```text
期望 = 胜率 × 平均盈利 - 失败率 × 平均亏损 - 交易成本
```

用 R 倍数表示：

```text
E = p × b - (1 - p) × a - c
```

其中：

- `E`：单笔交易期望，单位是 R；
- `p`：胜率，由回测或实盘记录统计出来；
- `b`：平均盈利，单位是 R，由回测或实盘记录统计出来；
- `a`：平均亏损倍数，等于 [实际平均亏损](01_Positive_Expectancy_and_Risk_Constraints/Actual_Average_Loss.md) / `TradeR`；
- `c`：手续费、滑点、冲击成本，单位是 R。

当平均亏损控制在 `1R` 时：

```text
E = p × b - (1 - p) - c
```

正期望条件：

```text
E > 0
```

---

## 6. 最低胜率要求

在平均亏损为 `1R` 的情况下：

```text
E = p × b - (1 - p) - c
```

要让 `E > 0`，需要满足：

```text
p > (1 + c) / (b + 1)
```

示例：

| 平均盈利 b | 成本 c | 最低胜率 p |
|---:|---:|---:|
| 1.5R | 0.10R | 44.0% |
| 2.0R | 0.10R | 36.7% |
| 2.5R | 0.10R | 31.4% |
| 3.0R | 0.10R | 27.5% |
| 2.0R | 0.20R | 40.0% |
| 2.5R | 0.20R | 34.3% |
| 3.0R | 0.20R | 30.0% |

结论：

> 盈亏比越高，对胜率要求越低；成本越高，对胜率要求越高。

---

## 7. 成本约束

小资金账户最怕成本吞噬 R。

成本包括：

1. 手续费；
2. 滑点；
3. 买卖价差；
4. 冲击成本；
5. 无法按计划价格成交的损耗。

成本统一换算成 R：

```text
c = 单笔交易总成本 / TradeR
```

约束规则：

```text
c <= c_max
```

初始建议：

```text
c_max = 0.20R
```

如果预估成本超过 `0.20R`，正期望难度明显上升。

---

## 8. 单笔风险约束

单笔风险由账户权益决定。

```text
AccountR = 账户权益 × 单笔风险比例
```

初始建议：

```text
单笔风险比例 = 0.5% 到 1.0%
```

示例：

| 账户权益 | 0.5% 风险 | 1.0% 风险 |
|---:|---:|---:|
| 20,000 | 100 | 200 |
| 50,000 | 250 | 500 |
| 100,000 | 500 | 1,000 |

核心约束：

> `TradeR` 不能超过 `AccountR`。

---

## 9. 手数计算

先计算一手计划风险：

```text
一手风险 = |入场价 - 止损价| × 合约乘数 + 单手预估成本
```

再计算允许手数：

```text
允许手数 = floor(AccountR / 一手风险)
```

如果：

```text
允许手数 < 1
```

则放弃该笔交易。

---

## 10. 每日风险约束

单日亏损需要有硬上限。

初始建议：

```text
每日最大亏损 = 2 × AccountR
```

触发条件：

1. 当天累计亏损达到 `2 × AccountR`；
2. 当天连续亏损达到 2 到 3 笔；
3. 当天滑点明显异常；
4. 当天亏损明显偏离预期。

触发后：

```text
当天停止交易
```

目的：

> 防止小亏变成连续失控亏损。

---

## 11. 连续亏损约束

正期望系统也一定会遇到连续亏损。

如果胜率为 `p`，失败率为：

```text
q = 1 - p
```

连续亏损 `n` 笔的概率是：

```text
P = q^n
```

例如胜率 35%，失败率 65%：

| 连续亏损笔数 | 概率 |
|---:|---:|
| 3 | 27.46% |
| 5 | 11.60% |
| 8 | 3.19% |
| 10 | 1.35% |

这说明：

> 低胜率、高盈亏比系统必须能承受连续亏损。

初始约束：

```text
账户必须能承受 10 × AccountR 的连续亏损
```

如果账户无法承受 10R 连亏，则单笔风险比例设置过大。

---

## 12. 最大回撤约束

系统需要设置最大回撤边界。

初始建议：

```text
账户最大允许回撤 = 10% 到 15%
```

触发后：

1. 停止实盘交易；
2. 复盘交易记录；
3. 检查胜率、盈亏比、成本是否偏离模型；
4. 必要时降低单笔风险；
5. 如果模型失效，停止该系统。

最大回撤约束的目的不是优化收益，而是防止账户不可恢复。

---

## 13. 系统最低可接受条件

SmallFuturesLab 的规则组合至少要满足以下条件：

| 项目 | 初始要求 |
|---|---:|
| 平均亏损 a | ≤ 1R |
| 平均盈利 b | ≥ 2R |
| 胜率 p | 高于最低胜率要求 |
| 成本 c | ≤ 0.20R |
| `TradeR` | ≤ `AccountR` |
| 每日最大亏损 | 2 × AccountR |
| 连续亏损暂停 | 2 到 3 笔 |
| 最大回撤警戒 | 10% |
| 最大回撤停止 | 15% |

示例一：

```text
E = 0.35 × 2 - 0.65 - 0.20
E = -0.15R
```

结论：

> 平均盈利 2R、胜率 35%、成本 0.20R，这套组合不合格。

需要至少满足下面之一：

1. 提高胜率；
2. 提高平均盈利；
3. 降低交易成本；
4. 降低平均亏损。

示例二：

```text
E = 0.40 × 2 - 0.60 - 0.15
E = 0.05R
```

示例三：

```text
E = 0.35 × 2.5 - 0.65 - 0.15
E = 0.075R
```

## 14. 具体算例

本例展示模型从输入到输出的完整推算过程。

### 14.1 参数来源说明

本算例中的参数分为四类：

| 类型 | 含义 |
|---|---|
| 外部配置参数 | 使用者主动配置的参数。 |
| 单笔计划参数 | 某一笔交易计划中填写的参数。 |
| 统计估计参数 | 由回测或实盘记录统计出来的参数。 |
| 模型计算结果 | 根据公式推算出来的结果。 |

原则：

> 配置项可以手动设置；统计项必须来自数据；计算结果不能手填覆盖。

### 14.2 外部配置参数

账户参数：

| 参数 | 数值 | 来源 |
|---|---:|---|
| 账户权益 | 50,000 | 外部配置 |
| 单笔风险比例 | 0.5% | 外部配置 |
| 每日最大亏损 | 2 × AccountR | 外部配置 |
| 连续亏损暂停 | 2 笔 | 外部配置 |
| 最大回撤警戒 | 10% | 外部配置 |
| 最大回撤停止 | 15% | 外部配置 |
| 成本上限 c_max | 0.20R | 外部配置 |

期望约束：

| 参数 | 数值 | 来源 |
|---|---:|---|
| 最低平均盈利 b | 2.5R | 外部配置 |
| 平均亏损目标 a | 1R | 风险约束 |

说明：

```text
平均亏损目标 a = 1R
```

这不是随意填写的参数，而是系统通过止损规则和执行纪律要达成的风险约束。

### 14.3 统计估计参数

| 参数 | 数值 | 来源 |
|---|---:|---|
| 胜率 p | 35% | 回测或实盘记录统计 |
| 平均盈利 b | 2.5R | 回测或实盘记录统计 |
| 平均亏损 a | 1R | 回测或实盘记录统计 |

说明：

```text
p 不能主观填写。
```

它只能来自历史样本统计。没有样本时，只能作为待验证假设，不能当成系统结论。

### 14.4 单笔计划参数

| 参数 | 数值 | 来源 |
|---|---:|---|
| 入场价 | 3000 | 单笔计划 |
| 止损价 | 2980 | 单笔计划 |
| 合约乘数 | 10 | 单笔计划 |
| 单手预估成本 | 20 | 单笔计划 |

### 14.5 账户风险上限

```text
AccountR = 50,000 × 0.5%
AccountR = 250
```

来源：模型计算结果。

### 14.6 一手风险

```text
价格止损风险 = |3000 - 2980| × 10
价格止损风险 = 200
```

```text
一手总风险 = 价格止损风险 + 单手预估成本
一手总风险 = 200 + 20
一手总风险 = 220
```

```text
一手 TradeR = 220
```

来源：模型计算结果。

### 14.7 手数计算

```text
允许手数 = floor(250 / 220)
允许手数 = 1
```

输出约束：

```text
最大手数 = 1 手
```

实际单笔计划风险：

```text
TradeR = 220 × 1
TradeR = 220
```

实际单笔风险比例：

```text
实际单笔风险比例 = 220 / 50,000
实际单笔风险比例 = 0.44%
```

该风险低于账户设定的 `0.5%` 上限。

来源：模型计算结果。

### 14.8 成本换算

```text
c = 单手预估成本 / TradeR
c = 20 / 220
c = 0.09R
```

成本上限：

```text
c_max = 0.20R
```

输出约束：

```text
0.09R <= 0.20R
```

成本合格。

来源：模型计算结果。

### 14.9 正期望推算

使用公式：

```text
E = p × b - (1 - p) × a - c
```

代入参数：

```text
E = 0.35 × 2.5 - 0.65 × 1 - 0.09
E = 0.135R
```

换算成金额：

```text
长期单笔理论期望金额 = 0.135 × 220
长期单笔理论期望金额 = 29.70
```

输出结果：

```text
单笔期望 = +0.135R
长期单笔理论期望金额 = +29.70
```

该金额表示长期统计平均值，不是单笔止盈目标。

来源：模型计算结果。

### 14.10 最低胜率反推

在 `a = 1R` 时，使用公式：

```text
p > (1 + c) / (b + 1)
```

代入参数：

```text
p > (1 + 0.09) / (2.5 + 1)
p > 1.09 / 3.5
p > 31.14%
```

当前统计胜率：

```text
p = 35%
```

输出约束：

```text
35% > 31.14%
```

胜率满足最低要求。

来源：模型计算结果。

### 14.11 每日亏损约束

账户层每日最大亏损：

```text
每日最大亏损 = 2 × AccountR
每日最大亏损 = 2 × 250
每日最大亏损 = 500
```

以本笔交易的实际风险计算：

```text
连续亏损 2 笔 = 220 × 2 = 440
连续亏损 3 笔 = 220 × 3 = 660
```

输出约束：

```text
最多允许连续亏损 2 笔
第 3 笔前必须停止
```

来源：模型计算结果。

### 14.12 连续亏损承受能力

账户需要能承受 `10 × AccountR` 连续亏损。

```text
10 × AccountR = 10 × 250
10 × AccountR = 2,500
```

占账户权益比例：

```text
2,500 / 50,000 = 5%
```

输出约束：

```text
10 × AccountR 连续亏损 = 5% 回撤
```

该数值低于 `10%` 最大回撤警戒线。

来源：模型计算结果。

### 14.13 最大回撤约束

```text
最大回撤警戒 = 50,000 × 10% = 5,000
最大回撤停止 = 50,000 × 15% = 7,500
```

输出约束：

```text
账户亏损达到 5,000：进入警戒
账户亏损达到 7,500：停止系统
```

来源：模型计算结果。

### 14.15 反例

如果止损距离扩大为 30 点，其他参数不变：

```text
一手风险 = 30 × 10 + 20
一手风险 = 320
```

```text
允许手数 = floor(250 / 320)
允许手数 = 0
```

输出结果：

```text
不交易
```

原因：

```text
一手风险已经超过账户单笔风险上限
```

## 15. 规则验收条件

任何规则组合都必须先通过这几个问题：

1. 平均亏损是否能控制在 `1R` 以内；
2. 平均盈利是否能达到最低要求；
3. 成本是否低于上限；
4. 胜率是否高于最低要求；
5. 连续亏损是否在账户承受范围内；
6. 最大回撤是否在允许范围内；
7. 实际结果是否持续偏离模型。

不满足这些条件，规则组合无效。

---

## 16. 结论

SmallFuturesLab 的第一层规则是：

> 先定义正期望条件，再定义风险约束。

正期望模型决定系统是否值得研究。

风险约束决定账户能不能活到优势兑现。

核心顺序：

```text
区分可控与不可控 → 正期望条件 → 风险约束 → 执行记录 → 验证修正
```
