# 建立正期望与风险约束 v0.1

文件：`docs/01_Positive_Expectancy_and_Risk_Constraints.md`

## 1. 文档目的

本文档解决两个问题：

1. 一套交易规则在数学上怎样才有长期正期望；
2. 小资金账户应该如何从正期望模型反推风险约束。

本文件不定义具体入场形态，也不选择具体品种。

---

## 2. 核心思想

交易系统不能先从“想买什么”开始，而要先从“怎样才可能长期不亏”开始。

SmallFuturesLab 的第一步是：

> 先定义正期望条件，再定义风险边界。

也就是说：

1. 先计算一套规则需要什么胜率、盈亏比和成本水平；
2. 再确定单笔亏损、每日亏损、连续亏损和暂停规则；
3. 最后才进入品种过滤和行情结构选择。

---

## 3. R 的定义

系统统一使用 `R` 来表示风险单位。

```text
1R = 单笔交易的计划最大亏损
```

对期货交易来说：

```text
1R = |入场价 - 止损价| × 合约乘数 × 手数
```

如果考虑滑点和手续费，则使用：

```text
1R_total = 价格止损风险 + 预估交易成本
```

为了方便建模，后续默认：

```text
平均亏损 = 1R
```

如果实际执行中平均亏损长期大于 `1R`，说明止损执行有问题，系统需要重新评估。

---

## 4. 正期望模型

最基础的期望公式是：

```text
期望 = 胜率 × 平均盈利 - 失败率 × 平均亏损 - 交易成本
```

用 R 倍数表示：

```text
E = p × b - (1 - p) × a - c
```

其中：

- `E`：单笔交易期望，单位是 R；
- `p`：胜率；
- `b`：平均盈利，单位是 R；
- `a`：平均亏损，单位是 R；
- `c`：手续费、滑点、冲击成本，单位是 R。

如果系统把平均亏损控制在 `1R`，公式简化为：

```text
E = p × b - (1 - p) - c
```

正期望条件：

```text
E > 0
```

---

## 5. 最低胜率要求

在平均亏损为 `1R` 的情况下：

```text
E = p × b - (1 - p) - c
```

要让 `E > 0`，需要满足：

```text
p > (1 + c) / (b + 1)
```

示例：

| 平均盈利 b | 成本 c | 最低胜率 p |
|---:|---:|---:|
| 1.5R | 0.10R | 44.0% |
| 2.0R | 0.10R | 36.7% |
| 2.5R | 0.10R | 31.4% |
| 3.0R | 0.10R | 27.5% |
| 2.0R | 0.20R | 40.0% |
| 2.5R | 0.20R | 34.3% |
| 3.0R | 0.20R | 30.0% |

结论：

> 盈亏比越高，对胜率要求越低；成本越高，对胜率要求越高。

---

## 6. 成本约束

小资金账户最怕成本吞噬 R。

成本包括：

1. 手续费；
2. 滑点；
3. 买卖价差；
4. 冲击成本；
5. 无法按计划价格成交的损耗。

成本统一换算成 R：

```text
c = 单笔交易总成本 / 1R
```

约束规则：

```text
c <= c_max
```

初始建议：

```text
c_max = 0.20R
```

如果一笔交易的预估成本超过 `0.20R`，这笔交易不值得做。

原因很简单：

> 成本越接近 1R，正期望越难成立。

---

## 7. 单笔风险约束

单笔风险来自账户权益。

```text
单笔最大亏损金额 = 账户权益 × 单笔风险比例
```

初始建议：

```text
单笔风险比例 = 0.5% 到 1.0%
```

例如：

| 账户权益 | 0.5% 风险 | 1.0% 风险 |
|---:|---:|---:|
| 20,000 | 100 | 200 |
| 50,000 | 250 | 500 |
| 100,000 | 500 | 1,000 |

小资金账户原则：

> 如果一手合约的止损金额已经超过单笔风险上限，则该交易不能做。

---

## 8. 手数计算

先计算一手合约的计划风险：

```text
一手风险 = |入场价 - 止损价| × 合约乘数 + 单手预估成本
```

再计算允许手数：

```text
允许手数 = floor(单笔最大亏损金额 / 一手风险)
```

如果：

```text
允许手数 < 1
```

则不交易。

这条规则非常重要：

> 小资金不能因为想交易，就强行交易风险过大的品种。

---

## 9. 每日风险约束

单日亏损需要有硬上限。

初始建议：

```text
每日最大亏损 = 2R
```

触发条件：

1. 当天累计亏损达到 `2R`；
2. 当天连续亏损达到 2 到 3 笔；
3. 当天滑点明显异常；
4. 当天行情反复扫损，交易结构失效。

触发后：

```text
当天停止交易
```

目的：

> 防止小亏变成情绪化连续亏损。

---

## 10. 连续亏损约束

即使系统有正期望，也一定会遇到连续亏损。

如果胜率为 `p`，失败率为：

```text
q = 1 - p
```

连续亏损 `n` 笔的概率是：

```text
P = q^n
```

例如胜率 35%，失败率 65%：

| 连续亏损笔数 | 概率 |
|---:|---:|
| 3 | 27.46% |
| 5 | 11.60% |
| 8 | 3.19% |
| 10 | 1.35% |

这说明：

> 低胜率、高盈亏比系统必须能承受连续亏损。

初始约束：

```text
账户必须能承受 10R 连续亏损
```

如果账户无法承受 10R 连亏，则单笔 R 设置过大。

---

## 11. 最大回撤约束

系统需要设置最大回撤边界。

初始建议：

```text
账户最大允许回撤 = 10% 到 15%
```

触发后：

1. 停止实盘交易；
2. 复盘交易记录；
3. 检查胜率、盈亏比、成本是否偏离模型；
4. 必要时降低单笔风险；
5. 如果模型失效，停止该策略。

最大回撤约束的目的不是优化收益，而是防止账户不可恢复。

---

## 12. 系统最低可接受条件

SmallFuturesLab 的候选交易结构，至少要满足以下条件：

| 项目 | 初始要求 |
|---|---:|
| 平均亏损 | ≤ 1R |
| 平均盈利 | ≥ 2R |
| 胜率 | ≥ 35% |
| 成本 | ≤ 0.20R |
| 单笔风险 | 0.5% 到 1.0% 账户权益 |
| 每日最大亏损 | 2R |
| 连续亏损暂停 | 2 到 3 笔 |
| 最大回撤警戒 | 10% |
| 最大回撤停止 | 15% |

按这个初始模型计算：

```text
E = 0.35 × 2 - 0.65 - 0.20
E = -0.15R
```

所以：

> 如果平均盈利只有 2R，胜率 35%，成本 0.20R，这套系统不合格。

需要至少满足下面之一：

1. 提高胜率；
2. 提高平均盈利；
3. 降低交易成本；
4. 降低平均亏损。

例如：

```text
E = 0.40 × 2 - 0.60 - 0.15
E = 0.05R
```

或者：

```text
E = 0.35 × 2.5 - 0.65 - 0.15
E = 0.075R
```

---

## 13. 交易结构筛选标准

后续任何交易结构，都必须先通过这几个问题：

1. 止损能不能明确；
2. 止损距离能不能被账户承受；
3. 平均盈利有没有机会达到 2R 以上；
4. 成本是否低于 0.20R；
5. 胜率是否有机会达到最低要求；
6. 连续亏损后是否能自动暂停；
7. 是否适合日内交易，不依赖隔夜持仓。

不满足这些条件的结构，直接淘汰。

---

## 14. 当前结论

SmallFuturesLab 的第一层规则是：

> 不先找机会，先定义什么样的机会值得做。

正期望模型决定系统是否值得研究。

风险约束决定账户能不能活到优势兑现。

因此后续顺序固定为：

```text
正期望条件 → 风险约束 → 品种过滤 → 行情结构 → 回测验证 → 实盘记录
```
