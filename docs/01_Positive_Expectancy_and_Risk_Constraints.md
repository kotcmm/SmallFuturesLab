# 建立正期望与风险约束

## 1. 文档目的

本文档解决两个问题：

1. 一套交易规则在数学上怎样才有长期正期望；
2. 小资金账户应该如何从正期望模型反推风险约束。

---

## 2. 核心思想

SmallFuturesLab 的第一步是：

> 先定义正期望条件，再定义风险边界。

需要先确定：

1. 胜率、盈亏比和成本之间的关系；
2. 单笔亏损、每日亏损和连续亏损的上限；
3. 账户在亏损序列中的生存能力。

---

## 3. 可控制与不可控制

交易结果由可控制部分和不可控制部分共同决定。

不可控制部分不能靠意愿解决，只能通过规则承受。

可控制部分必须写成硬约束，并严格执行。

### 3.1 不可控制部分

| 项目 | 说明 |
|---|---|
| 单笔交易结果 | 下一笔交易赚钱还是亏钱不可提前确定。 |
| 短期胜率 | 连续几天、几十笔交易的胜率可能大幅波动。 |
| 连续亏损 | 正期望系统也可能出现连续亏损。 |
| 极端波动 | 突发波动可能导致滑点扩大。 |
| 成交价格 | 实际成交价可能偏离计划价。 |
| 市场状态 | 市场可能长时间不提供适合系统的机会。 |

这些变量只能估计，不能控制。

### 3.2 可控制部分

| 项目 | 约束方向 |
|---|---|
| 是否交易 | 不满足条件就不交易。 |
| 单笔风险 | 每笔亏损限制在预设 R 内。 |
| 仓位手数 | 根据账户权益和止损距离计算。 |
| 成本上限 | 成本超过阈值则放弃。 |
| 每日亏损 | 达到上限后停止。 |
| 连续亏损 | 达到暂停条件后停止。 |
| 最大回撤 | 达到警戒线后降风险，达到停止线后停用。 |
| 执行记录 | 每笔交易必须记录结果。 |

系统建设原则：

> 不试图控制市场，只控制自己暴露给市场的风险。

---

## 4. R 的定义

系统统一使用 `R` 表示风险单位。

```text
1R = 单笔交易的计划最大亏损
```

对期货交易来说：

```text
1R = |入场价 - 止损价| × 合约乘数 × 手数
```

考虑手续费和滑点后：

```text
1R_total = 价格止损风险 + 预估交易成本
```

后续默认：

```text
平均亏损 = 1R
```

如果实际平均亏损长期大于 `1R`，说明止损执行或成本控制有问题。

---

## 5. 正期望模型

基础公式：

```text
期望 = 胜率 × 平均盈利 - 失败率 × 平均亏损 - 交易成本
```

用 R 倍数表示：

```text
E = p × b - (1 - p) × a - c
```

其中：

- `E`：单笔交易期望，单位是 R；
- `p`：胜率；
- `b`：平均盈利，单位是 R；
- `a`：平均亏损，单位是 R；
- `c`：手续费、滑点、冲击成本，单位是 R。

当平均亏损控制在 `1R` 时：

```text
E = p × b - (1 - p) - c
```

正期望条件：

```text
E > 0
```

---

## 6. 最低胜率要求

在平均亏损为 `1R` 的情况下：

```text
E = p × b - (1 - p) - c
```

要让 `E > 0`，需要满足：

```text
p > (1 + c) / (b + 1)
```

示例：

| 平均盈利 b | 成本 c | 最低胜率 p |
|---:|---:|---:|
| 1.5R | 0.10R | 44.0% |
| 2.0R | 0.10R | 36.7% |
| 2.5R | 0.10R | 31.4% |
| 3.0R | 0.10R | 27.5% |
| 2.0R | 0.20R | 40.0% |
| 2.5R | 0.20R | 34.3% |
| 3.0R | 0.20R | 30.0% |

结论：

> 盈亏比越高，对胜率要求越低；成本越高，对胜率要求越高。

---

## 7. 成本约束

小资金账户最怕成本吞噬 R。

成本包括：

1. 手续费；
2. 滑点；
3. 买卖价差；
4. 冲击成本；
5. 无法按计划价格成交的损耗。

成本统一换算成 R：

```text
c = 单笔交易总成本 / 1R
```

约束规则：

```text
c <= c_max
```

初始建议：

```text
c_max = 0.20R
```

如果预估成本超过 `0.20R`，正期望难度明显上升。

---

## 8. 单笔风险约束

单笔风险由账户权益决定。

```text
单笔最大亏损金额 = 账户权益 × 单笔风险比例
```

初始建议：

```text
单笔风险比例 = 0.5% 到 1.0%
```

示例：

| 账户权益 | 0.5% 风险 | 1.0% 风险 |
|---:|---:|---:|
| 20,000 | 100 | 200 |
| 50,000 | 250 | 500 |
| 100,000 | 500 | 1,000 |

核心约束：

> 单笔计划亏损不能超过账户可承受上限。

---

## 9. 手数计算

先计算一手计划风险：

```text
一手风险 = |入场价 - 止损价| × 合约乘数 + 单手预估成本
```

再计算允许手数：

```text
允许手数 = floor(单笔最大亏损金额 / 一手风险)
```

如果：

```text
允许手数 < 1
```

则放弃该笔交易。

---

## 10. 每日风险约束

单日亏损需要有硬上限。

初始建议：

```text
每日最大亏损 = 2R
```

触发条件：

1. 当天累计亏损达到 `2R`；
2. 当天连续亏损达到 2 到 3 笔；
3. 当天滑点明显异常；
4. 当天亏损明显偏离预期。

触发后：

```text
当天停止交易
```

目的：

> 防止小亏变成连续失控亏损。

---

## 11. 连续亏损约束

正期望系统也一定会遇到连续亏损。

如果胜率为 `p`，失败率为：

```text
q = 1 - p
```

连续亏损 `n` 笔的概率是：

```text
P = q^n
```

例如胜率 35%，失败率 65%：

| 连续亏损笔数 | 概率 |
|---:|---:|
| 3 | 27.46% |
| 5 | 11.60% |
| 8 | 3.19% |
| 10 | 1.35% |

这说明：

> 低胜率、高盈亏比系统必须能承受连续亏损。

初始约束：

```text
账户必须能承受 10R 连续亏损
```

如果账户无法承受 10R 连亏，则单笔 R 设置过大。

---

## 12. 最大回撤约束

系统需要设置最大回撤边界。

初始建议：

```text
账户最大允许回撤 = 10% 到 15%
```

触发后：

1. 停止实盘交易；
2. 复盘交易记录；
3. 检查胜率、盈亏比、成本是否偏离模型；
4. 必要时降低单笔风险；
5. 如果模型失效，停止该系统。

最大回撤约束的目的不是优化收益，而是防止账户不可恢复。

---

## 13. 系统最低可接受条件

SmallFuturesLab 的规则组合至少要满足以下条件：

| 项目 | 初始要求 |
|---|---:|
| 平均亏损 | ≤ 1R |
| 平均盈利 | ≥ 2R |
| 胜率 | ≥ 35% |
| 成本 | ≤ 0.20R |
| 单笔风险 | 0.5% 到 1.0% 账户权益 |
| 每日最大亏损 | 2R |
| 连续亏损暂停 | 2 到 3 笔 |
| 最大回撤警戒 | 10% |
| 最大回撤停止 | 15% |

按这个初始模型计算：

```text
E = 0.35 × 2 - 0.65 - 0.20
E = -0.15R
```

所以：

> 如果平均盈利只有 2R，胜率 35%，成本 0.20R，这套组合不合格。

需要至少满足下面之一：

1. 提高胜率；
2. 提高平均盈利；
3. 降低交易成本；
4. 降低平均亏损。

例如：

```text
E = 0.40 × 2 - 0.60 - 0.15
E = 0.05R
```

或者：

```text
E = 0.35 × 2.5 - 0.65 - 0.15
E = 0.075R
```

---

## 14. 规则验收条件

任何规则组合都必须先通过这几个问题：

1. 平均亏损是否能控制在 `1R` 以内；
2. 平均盈利是否能达到最低要求；
3. 成本是否低于上限；
4. 胜率是否高于最低要求；
5. 连续亏损是否在账户承受范围内；
6. 最大回撤是否在允许范围内；
7. 实际结果是否持续偏离模型。

不满足这些条件，规则组合无效。

---

## 15. 结论

SmallFuturesLab 的第一层规则是：

> 先定义正期望条件，再定义风险约束。

正期望模型决定系统是否值得研究。

风险约束决定账户能不能活到优势兑现。

核心顺序：

```text
区分可控与不可控 → 正期望条件 → 风险约束 → 执行记录 → 验证修正
```
