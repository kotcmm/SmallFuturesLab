# SmallFuturesLab

SmallFuturesLab 是一个独立的小资金期货交易研究项目。

本项目与 `TradingOS` / VSB 项目相互独立。

- `TradingOS` 继续研究 VSB（Volatility Squeeze Breakout，波动率收缩突破）长期主策略。
- `SmallFuturesLab` 专门研究 1 万到 2 万资金规模下，是否存在可执行、可控制风险、可复盘的小资金期货交易方法。

两个项目不共享策略规则，不互相迁就，不用一个项目的结论反向修改另一个项目。

---

## 1. 项目目标

SmallFuturesLab 的第一目标不是暴利，而是：

```text
生存
控制亏损
训练执行
积累真实交易样本
在不破坏账户的前提下寻找可持续优势
```

本项目研究的是：

```text
小资金账户在期货市场中，什么交易行为可以被账户承受；
哪些品种、周期、规则和风控方式可能适合 1 万到 2 万资金；
如何避免因为合约颗粒度、手续费、滑点、连续亏损导致账户快速损伤。
```

---

## 2. 项目不做什么

本项目明确不做：

```text
不追求短期暴利
不研究网格交易
不研究亏损摊平
不研究扛单回本
不研究重仓翻倍
不使用无法承受连续亏损的规则
不为了提高胜率而扩大亏损
不把 VSB 的市场结构规则直接搬进本项目
```

---

## 3. 当前阶段

当前阶段只做三件事：

```text
1. 定义小资金账户约束
2. 建立品种与周期筛选框架
3. 形成候选策略方向，但暂不冻结具体交易规则
```

当前不写实盘代码，不接交易接口，不做收益率优化。

## 5. 与 TradingOS 的关系

SmallFuturesLab 与 TradingOS 的关系如下：

| 项目 | 定位 | 当前目标 |
|---|---|---|
| TradingOS | VSB 长期主策略研究 | 市场结构识别与趋势右尾研究 |
| SmallFuturesLab | 小资金交易研究 | 1 万到 2 万账户可执行性研究 |

允许共享的是：

```text
工程纪律
测试习惯
风险意识
复盘方法
基础数据处理经验
```

禁止共享的是：

```text
策略规则
入场逻辑
止损逻辑
退出逻辑
参数体系
收益目标
```

---

## 6. 研究纪律

任何候选策略必须先回答：

```text
一手最小风险是多少？
连续亏损能否承受？
手续费和滑点是否会吞噬优势？
是否需要隔夜？
是否会诱导重仓、摊平或扛单？
是否适合当前账户规模？
```

如果一个方法无法通过账户约束检查，则不进入策略研究。
